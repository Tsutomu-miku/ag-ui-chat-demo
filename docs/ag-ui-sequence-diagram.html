合并算法 (context.ts)
1. 遍历 thread.messages → 加入 result
2. 遍历 incoming messages:
  • 跳过 knownMessageIds
  • 跳过重复 assistant toolCall
  • 追加新消息
3. 返回完整有序消息数组
hydratedMessages: [all history + new msg] (去重)
传给 LangGraph
runLangGraphAgent({ ...input, messages: hydratedMessages })
Agent 拿到完整上下文进行推理
Agent Run 完成后 — 持久化
persistHistory(threadId, inputMessages, events) → appendMessages()
持久化算法 (persistence.ts)
1. 输入消息去重后存入
2. 从 AG-UI events 重建 assistant 消息:
  TEXT_MESSAGE_* → content
  TOOL_CALL_* → toolCalls[]
  TOOL_CALL_RESULT → tool message
3. 跳过已存在的 message ID
4. 跳过重复的 assistant toolCall
5. appendMessages() 写入 store
onComplete → refreshActive() → GET /api/history/threads/:id
下次用户发消息时，后端已有完整历史
前端只需发新消息即可
